Introduction

    Thinking Like a Programmer

      1. Procedural Thinking

        Procedural thinking means being able to craft clear and precise instructions that a computer can follow, lacking ambiguity and very detailed.

      2. Abstract Thinking

        Abstract thinking is having the ability to find a similarity between things that may seem very different. This is basically like finding the "least common denominator" between the two things.

        This is really important because it allows programmers to design things to be reused, so that they can repeat that effort thousands of times over.

      3. Systems Thinking

        Systems thinking is the ability to break a big problem down into smaller chunks. This is important in programming because it allows the programmer to think of the big-picture and to come up with a roadmap to completion.

      4. Technological Empathy

        Technological Empathy is the same thing as empathizing with another human. It's understanding how the computer works and what its strengths and weaknesses are. This allows the programmer to make the right design choices when it comes time to decide what the computer should do (math) and what the computer should not do (decide who lives).

      5. Debugging

        Debugging is a process of finding and resolving issues with programs. Even the best programmers make mistakes, so it is important to be able to identify and resolve these bugs as they arise.

Stage 0

    Computers Are Stupid

      Humans create sentences which are designed for other humans to understand. They are lacking structure and make lots of assumptions that other humans can interpret. Computers can't fill in these assumptions and require a very structured language which gives them very detailed instructions on what to do.

    It's OK to Forget

      New programmers often struggle with trying to remember everything because in other learning environments you have to remember a lot of facts. Programming doesn't work like that, there's just too much to remember. The important thing is to be able to remember the big ideas, and know how to find the details that you need as you need them.

    Block Elements vs inline elements

      Block elements like <div> create extra stuff like spacing in an invisible block around the element. Inline elements, like <span>, do not have this effect.
